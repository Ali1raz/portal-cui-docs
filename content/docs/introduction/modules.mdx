# Features

## Student Application and Registration 

Prospective students submit applications through the portal. Clerks review and approve
or request revisions. Once admitted, students register for semesters and select subjects
within defined enrollment windows. Drop and late-drop deadlines are tracked per semester.

## Role-Based Access Control

Every user gets a scoped view based on their role. Students see only their own records.
Batch advisors see their department queue. HODs see their department. Accountants and
admins have broader access appropriate to their responsibilities. No role can call
another role's actions, even directly.


## Leave Management

Students submit leave requests tied to a specific subject and date. The system rejects
duplicates upfront. Requests move through a two-stage review chain: batch advisor first,
then HOD. At each stage the reviewer can approve, reject, or ask for more information,
which sends the request back to the student for revision. Every status change is logged.
Once a leave is approved, admin can retroactively correct the attendance record.

## Announcement System

HODs post announcements scoped to their department. Accountants can broadcast
portal-wide or filter by department, program, batch, or year. Authors can publish
immediately, save as a draft, or schedule for a future date. Scheduled announcements
publish automatically via a background job. Pinned announcements surface at the top
of the student feed.

## Fee Installment Management

Accountants define a base installment structure for each semester. Students see their
dues and can download a payment voucher on demand. If the default plan does not work,
a student can request a custom split of any unpaid installment. That request goes through
HOD review first, then accountant final approval. The total number of installments per
student is capped at three. Approved splits update the installment records immediately.

## Complaint Management

Students file complaints with a category, title, details, and an optional attachment.
Complaints start in the batch advisor queue. The BA can request more information,
reject, or forward to the HOD. The HOD can resolve the complaint, reject it, or
reassign it to a different administrative department entirely. Every action taken by
every actor is written to an immutable audit log, so the full history of a complaint
is always reconstructable.

## Attendance Tracking

Professors create an attendance record per lecture, marking each enrolled student as
present, absent, or on leave. Students can see their attendance percentage per subject.
The leave module ties into attendance: an approved leave request allows admin to flip
an absence to leave without the professor needing to re-mark anything.
